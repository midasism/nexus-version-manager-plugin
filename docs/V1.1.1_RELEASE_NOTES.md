# v1.1.1 版本发布说明

## 🎯 本次更新

本版本主要修复了上传时间显示和用户体验问题。

## ✅ 已修复的问题

### 1. 移除"仓库"列 ✅

**改进原因：**
- 仓库信息对日常使用价值不大
- 简化表格，突出关键信息
- 为上传时间列腾出更多空间

**修改前：**
```
| 模块名 | 版本号 | 类型 | 仓库 | 上传时间 |
```

**修改后：**
```
| 模块名 | 版本号 | 类型 | 上传时间 |
```

### 2. 修复上传时间显示 ✅

**问题描述：**
- 上传时间列始终显示为 "-"
- 无法看到版本的发布时间

**根本原因：**
- Nexus API返回的时间字段名是 `lastBlobUpdated`（而不是预期的 `lastModified`）
- 时间格式是ISO 8601字符串（如 `"2026-01-28T05:04:20.555Z"`），而不是long型时间戳

**解决方案：**
1. 正确读取 `lastBlobUpdated` 字段
2. 实现ISO 8601日期格式解析
3. 支持带毫秒和不带毫秒两种格式
4. 转换为本地时区显示

**修改的文件：**
- `NexusApiService.kt` - 新增 `parseIso8601ToTimestamp()` 方法

**现在的显示效果：**
```
| 模块名                | 版本号         | 类型 | 上传时间          |
| distinct-base-data-api | 2.6.1-SNAPSHOT | 快照 | 2026-01-28 13:04:20 |
| distinct-base-data-api | 2.6.0-SNAPSHOT | 快照 | 2026-01-27 10:30:15 |
```

### 3. 首次打开自动加载数据 ✅

**问题描述：**
- 首次打开工具窗口是空白的
- 需要手动点击"刷新所有API模块"按钮
- 用户体验不友好

**解决方案：**
- 工具窗口打开后自动触发刷新
- 延迟1秒执行，确保Maven项目已加载
- 使用后台线程，不阻塞UI

**修改的文件：**
- `NexusVersionToolWindow.kt` - 在 `getContent()` 方法中添加自动刷新逻辑

**用户体验改进：**
```
旧版：打开工具窗口 → 看到空白 → 手动点刷新 → 等待加载 → 看到数据
新版：打开工具窗口 → 自动加载 → 看到数据
```

## 📊 技术细节

### ISO 8601 日期解析

**Nexus API返回格式：**
```json
{
  "lastBlobUpdated": "2026-01-28T05:04:20.555Z"
}
```

**解析逻辑：**
```kotlin
private fun parseIso8601ToTimestamp(dateStr: String): Long {
    return try {
        // 首先尝试带毫秒的格式
        val format = SimpleDateFormat("yyyy-MM-dd'T'HH:mm:ss.SSS'Z'")
        format.timeZone = TimeZone.getTimeZone("UTC")
        format.parse(dateStr)?.time ?: 0L
    } catch (e: Exception) {
        // 如果失败，尝试不带毫秒的格式
        val format = SimpleDateFormat("yyyy-MM-dd'T'HH:mm:ss'Z'")
        format.timeZone = TimeZone.getTimeZone("UTC")
        format.parse(dateStr)?.time ?: 0L
    }
}
```

**显示格式：**
```kotlin
// VersionTableModel.kt
private val dateFormat = SimpleDateFormat("yyyy-MM-dd HH:mm:ss")
```

### 自动加载机制

**实现原理：**
```kotlin
SwingUtilities.invokeLater {
    Thread {
        Thread.sleep(1000)  // 延迟1秒
        ApplicationManager.getApplication().invokeLater {
            refreshAllApiModules()  // 自动刷新
        }
    }.start()
}
```

**为什么延迟1秒？**
1. 确保工具窗口UI完全初始化
2. 给Maven项目一些加载时间
3. 避免启动时的性能压力
