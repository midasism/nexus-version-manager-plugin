# 升级指南：从 v1.0.0 到 v1.1.0

## 🎯 主要改进

v1.1.0版本完全重构了用户体验，解决了构建插件后无法使用的关键问题，并添加了版本过滤功能。

## 📋 升级步骤

### 方法1：重新构建安装（推荐）

```bash
cd /Users/midasgao/code/distinct/nexus-version-manager-plugin

# 确保已初始化Gradle Wrapper
# 如果gradle-wrapper.jar不存在，执行：
gradle wrapper --gradle-version=8.5
# 或下载：
curl -o gradle/wrapper/gradle-wrapper.jar \
  https://repo1.maven.org/maven2/org/gradle/gradle-wrapper/8.5/gradle-wrapper-8.5.jar

# 构建新版本
./gradlew clean buildPlugin

# 生成的插件位于：
# build/distributions/nexus-version-manager-plugin-1.1.0.zip
```

然后在IDEA中安装：
```
1. Settings > Plugins
2. 点击已安装的"Nexus Version Manager"
3. 点击卸载（Uninstall）
4. 重启IDEA
5. Settings > Plugins > 齿轮图标 > Install Plugin from Disk
6. 选择 nexus-version-manager-plugin-1.1.0.zip
7. 重启IDEA
```

### 方法2：开发调试

```bash
cd nexus-version-manager-plugin
./gradlew runIde
```

这会启动一个带有最新插件的IDEA实例进行测试。

### 代码变化

如果你修改过插件代码，需要注意以下变更：

#### 1. NexusVersionToolWindow.kt
- 移除了 `moduleComboBox`
- 新增了版本过滤单选按钮
- `loadModules()` 被移除
- `refreshVersions()` 重命名为 `refreshAllApiModules()`
- 新增 `applyVersionFilter()` 方法

#### 2. VersionTableModel.kt
- 列数从4列增加到5列
- 新增"模块名"列（第0列）
- 其他列索引都后移了1位

#### 3. ProjectInfoService.kt
- 增强了错误处理
- 新增 `isMavenProject()` 方法
- 优化了 `getApiModules()` 方法，添加日志输出

#### 4. NexusApiService.kt
- 新增 `init {}` 块，启动时自动加载配置

## ⚠️ 注意事项

### 1. 配置迁移

配置文件会自动保留，无需重新配置。如果配置丢失，参考：

```
配置文件位置：
macOS: ~/Library/Application Support/JetBrains/<IDEA版本>/options/nexus-version-manager.xml
```

### 2. Maven项目要求

v1.1.0要求项目必须：
- 是标准的Maven项目
- 包含至少一个以`-api`结尾的模块
- Maven项目已完全加载

### 3. 性能影响

查询所有API模块比单个模块耗时更长：
- v1.0.0：查询1个模块约2-5秒
- v1.1.0：查询N个模块约2N-5N秒

如果项目有很多API模块（>10个），首次查询可能需要30秒以上。

### 4. 兼容性

| 平台 | v1.0.0 | v1.1.0 |
|------|--------|--------|
| IDEA 2023.1+ | ✅ | ✅ |
| IDEA 2022.x | ⚠️ 未测试 | ⚠️ 未测试 |
| Maven项目 | ✅ | ✅ |
| Gradle项目 | ❌ | ❌ |

## 🐛 已知问题修复

### 问题2：配置不持久化 ✅ 已修复

**v1.0.0问题：**
- 每次打开IDEA需要重新配置

**v1.1.0修复：**
- Service初始化时自动加载配置
- 配置文件正确持久化

### 问题3：无法区分不同模块 ✅ 已修复

**v1.0.0问题：**
- 如果多个模块有相同版本号，无法区分

**v1.1.0修复：**

## 📖 新功能使用教程

### 功能1：自动查询所有API模块

```
【旧版操作】
1. 选择模块1 → 刷新 → 查看版本
2. 选择模块2 → 刷新 → 查看版本
3. 选择模块3 → 刷新 → 查看版本
（每个模块都要手动操作）

【新版操作】
1. 点击"刷新所有API模块"
2. 等待查询完成
3. 一次看到所有模块的所有版本
```

### 功能2：版本过滤

```
【使用场景1】只看正式版本
问题：我只想知道哪些模块有最新的正式版本
操作：
1. 刷新后，点击"仅正式版本"单选按钮
2. 表格立即过滤，只显示正式版本
3. 快速找到最新的稳定版本

【使用场景2】查找快照版本
问题：我想知道哪些模块正在开发中
操作：
1. 点击"仅快照版本"
2. 看到所有正在开发的版本
3. 可以判断各模块的开发进度
```

### 功能3：模块名展示

```
【旧版问题】
表格中只有版本号，不知道来自哪个模块
如果查询多个模块，容易混淆

【新版改进】
表格第一列显示模块名
可以清楚看到：
- distinct-base-data-api 有哪些版本
- distinct-appointment-api 有哪些版本
- ...
```
