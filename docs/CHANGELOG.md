# 更新日志

## [1.1.0] - 2024-04-07

### 🎉 新功能

#### 1. 自动识别所有API模块
- **移除模块下拉框**：不再需要手动选择模块
- **一键查询**：点击"刷新所有API模块"按钮，自动查询所有以`-api`结尾的模块
- **批量展示**：在一个表格中展示所有API模块的版本信息
- **模块区分**：新增"模块名"列，方便识别不同模块的版本

#### 2. 版本过滤功能
- **全部版本**：显示所有版本（快照+正式）
- **仅正式版本**：只显示正式发布的版本
- **仅快照版本**：只显示快照版本
- **实时切换**：通过单选按钮快速切换过滤条件

### 🔧 优化改进

#### 1. 解决构建插件后看不到模块的问题
- **增强错误处理**：添加Maven项目状态检查
- **空值防护**：完善所有可能为空的字段处理
- **调试信息**：添加详细的日志输出，便于排查问题
- **友好提示**：当找不到模块时给出明确的提示信息

#### 2. 配置自动加载
- **启动加载**：插件启动时自动加载已保存的Nexus配置
- **无需重配**：安装后只需配置一次，以后自动加载

#### 3. UI改进
- **布局优化**：工具栏分两层，更清晰
  - 第一层：刷新按钮
  - 第二层：版本过滤选项
- **表格优化**：新增"模块名"列，列宽优化
- **状态提示**：更详细的状态信息，显示查询进度和结果统计

### 📋 新的表格结构

| 模块名 | 版本号 | 类型 | 仓库 | 上传时间 |
|--------|--------|------|------|----------|
| distinct-base-data-api | 2.5.6 | 正式 | releases | 2024-04-01 |
| distinct-appointment-api | 3.2.1 | 快照 | snapshots | 2024-04-07 |

### 🎯 使用场景

#### 场景1：查看所有API模块的最新版本
```
1. 打开工具窗口
2. 点击"刷新所有API模块"
3. 等待查询完成
4. 查看所有模块的版本信息
```

#### 场景2：只看正式版本
```
1. 刷新后，所有版本都会显示
2. 点击"仅正式版本"单选按钮
3. 表格立即过滤，只显示正式版本
```

#### 场景3：查找特定模块的快照版本
```
1. 点击"仅快照版本"
2. 在表格中按模块名排序（点击列标题）
3. 找到目标模块的快照版本
```

### ⚠️ 注意事项

1. **首次使用**：
   - 确保已在Settings中配置Nexus地址和Token
   - 确保Maven项目已完全加载

2. **性能考虑**：
   - 查询多个模块需要一定时间，请耐心等待
   - 每个模块的查询是串行的，避免服务器压力过大

3. **错误提示**：
   - 如果提示"未找到API模块"，检查项目是否有`-api`结尾的模块
   - 如果查询失败，检查Nexus配置和网络连接

### 🐛 已修复的问题

1. **构建插件后看不到模块**
   - 原因：Maven项目未完全初始化就尝试获取模块
   - 解决：添加Maven状态检查和空值防护

2. **配置不持久化**
   - 原因：Service未在初始化时加载配置
   - 解决：在NexusApiService初始化时加载已保存的配置

3. **无法区分不同模块的版本**
   - 原因：表格中缺少模块信息
   - 解决：增加"模块名"列

### 📦 升级方法

#### 方法1：重新构建安装
```bash
cd nexus-version-manager-plugin
./gradlew buildPlugin
```
然后在IDEA中：`Settings > Plugins > 卸载旧版本 > 安装新版本`

#### 方法2：开发调试
```bash
./gradlew runIde
```

### 💡 使用技巧

1. **快速过滤**：使用键盘快捷键快速选择过滤选项
2. **排序查看**：点击表格列标题可以按该列排序
3. **批量复制**：选中多行后右键复制（如果需要这个功能，可以后续添加）
