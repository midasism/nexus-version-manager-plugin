# v1.1.0 改进总结

## 🎯 解决的核心问题

### 1. 构建插件后无法使用 ✅ 已解决

**问题描述：**
```
./gradlew runIde        → ✅ 能看到模块和数据
./gradlew buildPlugin   → ❌ 安装后看不到模块
```

**根本原因：**
- Maven项目在插件初始化时尚未完全加载
- ProjectInfoService获取模块时MavenProjectsManager返回空列表
- 缺少Maven项目状态检查和空值防护

**解决方案：**
1. **增强ProjectInfoService**
   - 添加 `isMavenizedProject` 状态检查
   - 完善空值检查（groupId、artifactId、version）
   - 添加详细的调试日志输出
   - 优化错误处理和友好提示

2. **优化初始化流程**
   - NexusApiService在init块中加载配置
   - 延迟获取service实例，确保完全加载
   - 添加Maven项目加载等待提示

3. **改进用户体验**
   - 当找不到模块时，给出明确的操作指引
   - 区分"Maven未加载"和"无API模块"两种情况
   - 提供详细的错误信息和解决方案

### 2. 只显示API模块，移除下拉框 ✅ 已实现

**旧版设计：**
```
┌────────────────────────────────┐
│ 选择模块: [下拉框▼] [刷新]    │
└────────────────────────────────┘
需要用户手动选择每个模块
```

**新版设计：**
```
┌────────────────────────────────┐
│ [刷新所有API模块]              │
└────────────────────────────────┘
自动识别并查询所有-api结尾的模块
```

**实现细节：**
- 使用 `ProjectInfoService.getApiModules()` 自动过滤
- 批量查询所有API模块，显示进度条
- 在表格中增加"模块名"列区分不同模块
- 状态栏显示："共找到 X 个版本，来自 Y 个API模块"

### 3. 快照和正式版本分开显示 ✅ 已实现

**旧版：**
```
所有版本混在一起，无法快速过滤
```

**新版：**
```
┌─────────────────────────────────────┐
│ 显示: (•)全部版本                  │
│       ( )仅正式版本                │
│       ( )仅快照版本                │
└─────────────────────────────────────┘
```

**实现细节：**
- 使用单选按钮组（ButtonGroup）
- 实时过滤，无需重新查询
- 状态栏显示当前过滤结果统计
